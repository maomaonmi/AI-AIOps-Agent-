# CPU使用率异常排查流程 (SOP-0007)

## 现象描述
服务器CPU使用率持续超过阈值（通常90%），影响服务性能。

## 排查步骤

### 1. 确认CPU使用率
- 查询Prometheus: `avg by(instance)(rate(node_cpu_seconds_total{mode!="idle"}[5m])) * 100`
- 确认是单核还是多核飙高

### 2. 定位高CPU进程
```bash
top -bn1 -o %CPU | head -20
```
- 查看占用CPU最高的进程
- 记录PID和进程名

### 3. 分析进程详情
```bash
# 查看进程线程
top -Hp <PID>

# 查看线程堆栈
jstack <PID> > thread_dump.txt

# 查看GC情况
jstat -gcutil <PID> 1000 10
```

### 4. 常见原因
1. **GC频繁**: Java堆内存不足，频繁Full GC
2. **死循环**: 代码逻辑错误导致无限循环
3. **计算密集**: 大数据量处理、加密运算
4. **线程泄漏**: 创建大量线程未释放
5. **上下文切换**: 线程数过多导致频繁切换

### 5. 处理方案
- GC频繁: 参照OOM处理流程
- 死循环: 修复代码，发布热修复
- 计算密集: 优化算法，异步处理
- 线程泄漏: 修复线程池配置
- 上下文切换: 减少线程数，使用协程

## 监控告警配置
- 告警阈值: CPU > 90% 持续5分钟
- 通知方式: P1电话 + P2企微
